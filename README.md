# Algorithm_Study
파이썬 기초 문법 부터 자료구조 까지

## Ⅰ. 자료구조 개념 및 파이썬 활용법 (기초~심화)

문법 공부가 끝나면 아래 순서대로 개념을 잡고, 파이썬으로 어떻게 구현하는지 익혀야 합니다.

### 1단계: 선형 자료구조 (일렬로 나열된 데이터)

가장 기본이 되며, 파이썬에서는 기본 문법만으로도 쉽게 다룰 수 있습니다.

- **배열 (Array / List)**
    - **개념:** 데이터를 순서대로 저장하고 인덱스(번호)로 접근합니다.
    - **파이썬:** 그냥 `List([])`가 이 역할을 다 합니다. (크기도 자동 조절됨)
    - **체크포인트:** 2차원 리스트(행렬)를 만들고 다루는 법을 꼭 익혀야 합니다.
- **스택 (Stack)**
    - **개념:** '프링글스 통' 같습니다. 나중에 넣은 것이 먼저 나옵니다 (LIFO: Last In First Out).
    - **용도:** 웹 브라우저 '뒤로 가기', **DFS 알고리즘**, 복잡한 SQL 쿼리 파싱, 중첩된 JSON 데이터 처리.
    - **파이썬:** 별도 라이브러리 없이 List의 `.append()`와 `.pop()`만 쓰면 끝입니다.
- **큐 (Queue)**
    - **개념:** '놀이공원 줄서기' 같습니다. 먼저 줄 선 데이터가 먼저 나갑니다 (FIFO: First In First Out).
    - **용도:** 맛집 대기열, **BFS 알고리즘**.
    - **데이터 엔지니어링 관련:** **데이터 파이프라인(Kafka, RabbitMQ 등)**의 기본 원리입니다. 데이터가 순서대로 처리되지 않으면 발생하는 문제를 이해하는 데 필수적입니다.
    - **파이썬:** `collections` 모듈의 `deque`를 사용해야 속도가 빠릅니다. (일반 리스트는 느림)

### 2단계: 비선형 자료구조 (데이터 검색과 효율성 핵심)

여기서부터가 진짜 알고리즘 실력(효율성)을 가릅니다. 대용량 데이터 처리가 핵심인 데이터 엔지니어링에서 매우 중요합니다.

- **해시 테이블 (Hash Table / Dictionary) ⭐ (가장 중요)**
    - **개념:** Key(키)를 알면 Value(값)를 **단 한 번에** 즉시 찾을 수 있는 마법 같은 구조입니다.
    - **용도:** 중복 검사, 회원 정보 검색, 로그 데이터 분석.
    - **데이터 엔지니어링 관련:** 수억 건의 로그 데이터 중 특정 유저를 찾을 때, 리스트 탐색($O(N)$)은 시스템을 멈추게 하지만 해시($O(1)$)는 즉시 찾습니다.
    - **파이썬:** `Dictionary ({})`가 바로 이것입니다. 문법 공부할 때 딕셔너리를 완벽하게 파두어야 합니다.
- **집합 (Set)**
    - **개념:** 중복을 허용하지 않고 순서가 없는 데이터 모음입니다.
    - **용도:** **데이터 중복 제거(ETL 과정에서 필수)**.
    - **파이썬:** `set()`을 사용합니다. 파이썬의 집합 연산자(`&`, `|`, )를 활용하면 SQL의 JOIN이나 차집합을 코드 한 줄로 구현할 수 있습니다.

### 3단계: 심화 자료구조 (복잡한 문제 해결용)

- **힙 (Heap / Priority Queue)**
    - **개념:** 데이터 중 최댓값이나 최솟값을 빠르게 찾아내기 위한 완전 이진 트리입니다.
    - **파이썬:** `heapq` 모듈을 사용합니다.
- **트리 (Tree) & 그래프 (Graph)**
    - **개념:** 데이터가 계층적이거나(트리), 서로 복잡하게 연결된(그래프) 구조입니다.
    - **파이썬:** 별도 내장 타입은 없고, 리스트나 딕셔너리를 활용해 직접 구현합니다.

## Ⅱ. 핵심 문제집 상세 분석

### A. 기초 다지기: Python 배우기 & Solved.ac

1. **백준 'Python 배우기' (459~461번)**

    - **461번 (101~150):** 문자열 처리, 복잡한 반복문. 파이썬의 강력한 **문자열 슬라이싱** 숙달.
2. **Solved.ac CLASS 1~3**
    - **CLASS 1:** 문법 확인 수준 (Python 배우기와 겹침).
    - **CLASS 2:** **가장 중요.** 브루트 포스, 정렬, 스택/큐/이분 탐색 기초. "시간 초과"의 벽을 만나 효율성을 고민하게 됨.
    - **CLASS 3:** 첫 번째 고비. DFS/BFS, DP, 자료구조 심화 등장. 코테 합격권의 기초.

### B. 유형별 깊이 파기: 바킹독의 실전 알고리즘

C++ 기반이지만 구성이 완벽하여 파이썬 유저에게도 '바이블'로 통합니다. 논리적 사고력(알고리즘 근육)을 키웁니다.

- **0x03강 - 배열 (Array):**
    - **핵심:** 파이썬 `list`의 강력함에 의존하지 않고, 인덱스로 데이터에 빠르게 접근($O(1)$)하는 성질 이해.
    - **추천 문제:** `두 수의 합 (3273번)` (투 포인터/정렬 없이 배열 성질로만 풀어보기)
- **0x05강 - 스택 (Stack):**
    - **핵심:** 괄호 쌍 검사, 수식 계산 등 뒤에서부터 처리하는 로직.
    - **추천 문제:** `스택 (10828번)`, `제로 (10773번)`
- **0x06강 - 큐 (Queue):**
    - **핵심:** 순서대로 처리하기. 데이터 파이프라인의 원리 체험. `collections.deque` 마스터.
    - **추천 문제:** `큐 2 (18258번)`, `카드2 (2164번)`
- **0x09강 - BFS (너비 우선 탐색) ⭐:**
    - **핵심:** 이 문제집의 꽃. 그래프 탐색의 기초이자 코테 최다 빈출 유형. 난이도 곡선이 완만하여 학습하기 좋음.
    - **추천 문제:** `그림 (1926번)`, `미로 탐색 (2178번)`

### C. 데이터 효율성 특화: 백준 [집합과 맵]

"데이터 엔지니어의 실전 도구함". 로직보다는 자료구조 선택에 따른 효율성 차이를 체감하는 단계입니다.

- **핵심 의의:** 100만 개의 데이터에서 `if x in list`는 시스템을 멈추게 하지만, `if x in set`은 0.1초 만에 끝납니다.
- **주요 문제 분석:**
    - `숫자 카드 2 (10816번)`: 리스트 탐색($O(N)$) vs `Counter`/`Dictionary`($O(1)$) 차이 학습.
    - `듣보잡 (1764번)`: DB의 `INNER JOIN` 원리. 집합 연산자(`&`)로 한 줄 해결.
    - `나는야 포켓몬 마스터 이다솜 (1620번)`: 이름→번호, 번호→이름 양방향 검색 최적화를 위해 딕셔너리 2개 사용 (인덱싱 기초).

## Ⅲ. 최종 추천 학습 로드맵 (Step-by-Step)

무작정 순서대로 하는 것보다, 난이도를 계단식으로 올리는 최적의 순서입니다.

### Step 1: 파이썬 문법 장착 (워밍업)

- **참고:** 이 과정을 마치면 `CLASS 1`은 자동으로 클리어됩니다.

### Step 2: 선형 자료구조 & 해시 (데이터 다루기 기초)

- **할 것:** 백준 문제집 `Python 배우기 (101~150)` + `solved.ac CLASS 2`
- **병행 (필수 - 효율성 학습):**
    1. **자료구조 기초:** 바킹독 문제집 `0x03(배열)`, `0x05(스택)`, `0x06(큐)`
    2. **검색 효율화 (DE 강력 추천):** 백준 단계별로 풀어보기 - `[집합과 맵]`
- **이유:** `CLASS 2`부터 시간 초과가 발생합니다. 스택/큐로 로직을 풀고, 딕셔너리(해시)로 검색 속도를 획기적으로 줄이는 법을 익혀야 합니다. 특히 **[집합과 맵]**은 중복 제거와 고속 검색의 핵심이므로 필수입니다.

### Step 3: 알고리즘의 꽃 (그래프 & 트리 탐색)

- **할 것:** `solved.ac CLASS 3`
- **병행 (심화):** 바킹독 문제집 `0x09(BFS)`, `0x0A(DFS)`, `0x19(트리)`, `0x17(우선순위 큐)`
- **이유:** `CLASS 3`의 메인 테마는 DFS/BFS입니다. 문제를 풀다 막히면 바킹독 문제집의 해당 파트(특히 `0x09강 BFS`)를 집중적으로 풀어 유형을 체화하는 것이 빠릅니다. 트리는 그래프의 일종이므로 같이 학습하면 시너지가 납니다.
